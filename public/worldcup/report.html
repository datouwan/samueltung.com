FIFA World Cup 2026 · Project Report

Interactive World Cup site — functions & build learnings

A tour of what the site does, the data and architecture behind it, and the engineering lessons from getting live data fast and reliable.

← Back to the live site

The World Cup 2026 site is a single-page app served from a Cloudflare Worker. It pairs an interactive world map of all 48 qualified nations with live scores, standings, the knockout bracket, full squads, live line-ups, player & country stats, news, and a prediction game — in three languages.

Stack: Cloudflare Workers (edge API + static assets) · Leaflet map · vanilla JS, no framework · data proxied server-side so API keys stay private.

⚽What the site does

The interface is a sticky live bar on top of an interactive map, followed by a set of tabs.

🗺️ Interactive map

All 48 nations as flag markers, colored by confederation. A "Teams / Stadiums" toggle switches between teams and the 16 host stadiums. Live games draw animated flight-path arcs to the host stadium with a floating score card.

🔴 Live bar & scores

Sticky ticker of in-play games (accurate minute + score), next kickoffs, and a finished-scores strip. Goals flash when a score increments between polls.

👥 Live line-ups

Click a live card → both teams' Starting XI & subs with formation and coach. Falls back to full squads if the XI isn't posted yet. Players open their own card.

📋 Squads tab

Pick any nation to see its full World Cup squad grouped by position (GK / DF / MF / FW) with shirt number, club, age, and photo where available.

📊 Group stage

Live group tables (P/W/D/L/GD/Pts) with qualification highlighting and a "LIVE" badge on groups with a game in play.

📅 Schedule

Full fixture calendar overlaying live status/scores, auto-scrolling to the live or next game.

🏆 Knockout bracket

The full Round-of-32 → Final tree parsed from Wikipedia, filling in teams and scores as the tournament progresses.

🥉 3rd-place race

Live table of the best third-placed teams and their projected knockout slots.

🎲 Prediction game

A Monte-Carlo "bet" simulator and a dice mode that rolls a random tournament, with adjustable team boosts.

📈 Player & country stats

2026 top scorers plus all-time records (scorers, titles, appearances), with player search via Wikipedia.

📰 News

World Cup headlines merged and de-duplicated from several publisher RSS feeds (Guardian, BBC, Sky, CBS, talkSPORT).

🌐 i18n & theming

English / 中文 / Español, with country names localized to Chinese. Light/dark toggle; light theme locked to a warm "stone" palette.

🧰Data sources

football-data.orgfree

Group standings & schedule. No daily cap (10/min), so it's polled freely. Provides the "is anything live?" signal cheaply.

api-football (api-sports.io)Pro

Accurate live minute/score, line-ups, squads, player bios, goal events. Pro plan: 7,500/day, 300/min.

Wikipediafree

Bracket, all-time records, squads fallback, player/stadium photos & bios. Free and unlimited — used wherever possible to conserve the metered API.

🏗️Architecture notes

  • Edge-cached fan-in. Each upstream has its own cache key & TTL, so the page can poll often while upstreams are hit rarely — caching is shared across all visitors.
  • Keys stay server-side. The browser only ever talks to the Worker; API tokens live as Worker secrets.
  • Lazy & cheap. Expensive calls (line-ups, squads, player cards, events) fire only on interaction and are cached, so they never touch the page-load or polling path.
  • Graceful degradation. Every metered call has a free fallback (e.g. squads → Wikipedia; live score → football-data's delayed feed), so the site keeps working if the paid API is unavailable.
  • Cross-source name matching. A shared norm() + alias map reconciles spellings ("Korea Republic" ↔ "South Korea", "Ivory Coast" ↔ "Côte d'Ivoire") across every provider.

💡Engineering learnings

The cheap signal gates the expensive one

football-data (free, no daily cap) tells us whether anything is live; the metered api-football is only called when a game is actually in play. A free feed guarding a paid one keeps cost predictable.

Polling faster isn't free — pick the right payload

The biggest slowdown came from fetching api-football's global live=all feed (every live match on earth) every 10s. It tripped the 300/min limit and made requests slow. Switching to a World-Cup-only query by date — a handful of fixtures — made short polling both fast and safe.

Lesson: shrink the payload before shortening the interval.

Always put a timeout on a fetch

A single stalled request with no timeout left the page stuck on "Loading…" for minutes. An 8-second AbortController timeout (plus keeping the last-good data on screen) turned a freeze into a silent retry.

Match the cadence to the source

api-football refreshes live data roughly every 15s, so polling faster just re-fetches identical data. The poll interval was tuned to ~15s — near-real-time without waste.

Checkpoint before you experiment

When a change degraded performance, having the working tree stashed and a tagged commit (wc-stable) made rollback a one-liner. The line-up feature was then re-added on top and tagged separately (wc-lineups) so the two are trivial to compare.

Localize display, not data

Chinese names are applied only to visible text; the underlying English names stay as keys for lookups and cross-source matching — so translation never breaks logic.
